# <a name="prerequisites-for-certificate-profiles-in-configuration-manager"></a>Предварительные требования для профилей сертификатов в Configuration Manager

*Область применения: Configuration Manager (Current Branch)*


Профили сертификатов в Configuration Manager имеют внешние зависимости и зависимости в рамках продукта.  

## <a name="dependencies-external-to-configuration-manager"></a>Внешние зависимости Configuration Manager  

|Зависимость|Дополнительные сведения|  
|----------------|----------------------|  
|Выдающий центр сертификации (ЦС) предприятия с запущенными службами сертификатов Active Directory.<br /><br /> Для отзыва сертификатов учетной записи компьютера сервера сайта на верхнем уровне иерархии необходимы права *Выдавать сертификаты и управлять ими* для каждого шаблона сертификата, используемого профилем сертификата в Configuration Manager. Кроме того, можно предоставить диспетчеру сертификатов разрешения на предоставление разрешений по всем шаблонам сертификатов, используемым этим ЦС.<br /><br /> Поддерживается утверждение запросов на сертификаты диспетчером. Однако шаблоны сертификатов, используемые для выдачи сертификатов, должны быть настроены с использованием параметра **Предоставляется в запросе** для субъекта сертификата, чтобы система Configuration Manager могла автоматически указывать это значение.|Дополнительные сведения о службах сертификатов Active Directory см. в документации по Windows Server.<br /><br /> Для Windows Server 2012: [Обзор служб сертификатов Active Directory](https://go.microsoft.com/fwlink/p/?LinkId=286744)<br /><br /> Для Windows Server 2008: [Службы сертификатов Active Directory в Windows Server 2008](https://go.microsoft.com/fwlink/p/?LinkId=115018)|  
|Используйте сценарий PowerShell, чтобы проверить и при необходимости установить необходимые компоненты для службы роли службы регистрации сертификатов для сетевых устройств (NDES) и точки регистрации сертификатов Configuration Manager. <br /><br />|Файл инструкций readme_crp.txt находится в ConfigMgrInstallDir\cd.latest\SMSSETUP\POLICYMODULE\X64.<br /><br />Сценарий PowerShell Test-NDES-CRP-Prereqs.ps1 находится в одном каталоге с инструкциями. <br /><br /> Этот сценарий PowerShell нужно запустить локально на сервере NDES.|
|Служба роли службы регистрации сертификатов для сетевых устройств (NDES) для служб сертификатов Active Directory, запущенная на компьютере под управлением Windows Server 2012 R2.<br /><br /> Кроме того:<br /><br /> Номера TCP-портов, отличные от 443 (для HTTPS) и 80 (для HTTP), не поддерживаются для связи между клиентом и службой регистрации сертификатов для сетевых устройств.<br /><br /> Сервер, на котором запущена служба регистрации сертификатов для сетевых устройств, и выдающий ЦС должны находиться на разных компьютерах.|Configuration Manager взаимодействует со службой регистрации сертификатов для сетевых устройств в Windows Server 2012 R2 для создания и проверки запросов протокола SCEP.<br /><br /> Если сертификаты будут выдаваться для пользователей или устройств, которые подключаются через Интернет, таких как мобильные устройства под управлением Microsoft Intune, эти устройства должны иметь доступ к серверу, на котором запущена служба регистрации сертификатов для сетевых устройств, через Интернет. Например, установите сервер в сети периметра (также называемой демилитаризованной зоной или промежуточной подсетью).<br /><br /> Если между сервером, на котором запущена служба регистрации сертификатов для сетевых устройств, и выдающим ЦС имеется брандмауэр, его необходимо настроить, разрешив коммуникационный трафик (DCOM) между этими двумя серверами. Это требование к брандмауэру также распространяется на сервер, на котором запущен сервер сайта Configuration Manager и который выдает ЦС, чтобы система Configuration Manager могла отзывать сертификаты.<br /><br /> Если служба регистрации сертификатов для сетевых устройств настроена таким образом, чтобы требовать использование SSL, рекомендуем убедиться, что у подключающихся устройств есть доступ к списку отзыва сертификатов, с помощью которого эти устройства могут проверить сертификат сервера.<br /><br /> Дополнительные сведения о службе регистрации сертификатов для сетевых устройств в Windows Server 2012 R2 см. в статье [Using a Policy Module with the Network Device Enrollment Service (Использование модуля политики со службой регистрации сертификатов для сетевых устройств)](https://go.microsoft.com/fwlink/p/?LinkId=328657).|  
|Если выдающий ЦС работает под управлением Windows Server 2008 R2, этот сервер требует исправления для запросов SCEP на обновление.|Если на компьютере выдающего ЦС еще не установлено исправление, установите его. Дополнительные сведения см. в статье [об ошибке 2483564, когда запрос на обновление сертификата SCEP не выполняется в Windows Server 2008 R2, если сертификат является управляемым с помощью NDES](https://go.microsoft.com/fwlink/?LinkId=311945), в базе знаний Майкрософт.|  
|PKI-сертификат проверки подлинности клиента и экспортированный сертификат корневого ЦС.|Этот сертификат служит для проверки подлинности сервера, на котором запущена служба регистрации сертификатов для сетевых устройств, в Configuration Manager.<br /><br /> Дополнительные сведения см. в статье [Требования к PKI-сертификатам для Configuration Manager](../../core/plan-design/network/pki-certificate-requirements.md).|  
|Поддерживаемые операционные системы устройств.|Профили сертификатов можно развернуть на устройствах под управлением Windows 8.1, Windows RT 8.1 и Windows 10.|  

## <a name="configuration-manager-dependencies"></a>Зависимости Configuration Manager  

|Зависимость|Дополнительные сведения|  
|----------------|----------------------|  
|Роль системы сайта точки регистрации сертификатов|Перед использованием профилей сертификатов необходимо установить роль системы сайта точки регистрации сертификатов. Эта роль взаимодействует с базой данных Configuration Manager, сервером сайта Configuration Manager и модулем политики Configuration Manager.<br /><br /> Дополнительные сведения о системных требованиях для этой системной роли сайта и месте ее установки в иерархии см. в статье [Поддерживаемые конфигурации для System Center Configuration Manager](../../core/plan-design/configs/supported-configurations.md) в разделе о **системных требованиях сайта**.<br /><br /> Точку регистрации сертификатов нельзя устанавливать на том же сервере, на котором запущена служба регистрации сертификатов для сетевых устройств.|  
|Модуль политики Configuration Manager, установленный на сервере, на котором запущена служба роли "Служба регистрации сертификатов для сетевых устройств" для служб сертификатов Active Directory|Для развертывания профилей сертификатов необходимо установить модуль политики Configuration Manager. Его можно найти на установочном носителе Configuration Manager.|  
|Данные обнаружения|Значения субъекта сертификата и альтернативного имени субъекта указываются системой Configuration Manager и извлекаются из данных, собранных при обнаружении.<br /><br /> Для сертификатов пользователей: Обнаружение пользователей Active Directory<br /><br /> Для сертификатов компьютеров: обнаружение систем Active Directory и обнаружение сетевых ресурсов.|  
|Специальные разрешения системы безопасности для управления профилями сертификатов|Для управления параметрами доступа к ресурсам компании, такими как профили сертификатов, профили Wi-Fi и профили VPN, необходимо иметь следующие разрешения системы безопасности.<br /><br /> Для просмотра и контроля оповещений и отчетов по профилям сертификатов: **Создание**, **Удаление**, **Изменение**, **Изменение отчета**, **Чтение** и **Запуск отчета** для объекта **Оповещения**.<br /><br /> Для создания профилей сертификатов и управления ими: **Политика автора**, **Изменение отчета**, **Чтение** и **Запуск отчета** для объекта **Профиль сертификата**.<br /><br /> Для управления развертываниями профилей Wi-Fi, сертификатов и VPN: **Развертывание политик конфигурации**, **Изменение оповещения о состоянии клиента**, **Чтение** и **Чтение ресурса** для объекта **Коллекция**.<br /><br /> Для управления всеми политиками конфигураций: **Создание**, **Удаление**, **Изменение**, **Чтение** и **Настройка области безопасности** для объекта **Политика конфигурации**.<br /><br /> Для выполнения запросов, связанных с профилями сертификатов: разрешение на **Чтение** для объекта **Запрос**.<br /><br /> Для просмотра сведений о профилях сертификатов в консоли Configuration Manager: разрешение на **чтение** для объекта **Сайт**.<br /><br /> Для просмотра сообщений о состоянии для профилей сертификатов: разрешение на **Чтение** для объекта **Сообщения о состоянии**.<br /><br /> Для создания и изменения профиля сертификата доверенного ЦС: **Политика автора**, **Изменение отчета**, **Чтение** и **Запуск отчета** для объекта **Профиль сертификата доверенного ЦС**.<br /><br /> Для создания профилей VPN и управления ими: **Политика автора**, **Изменение отчета**, **Чтение** и **Запуск отчета** для объекта **Профиль VPN**.<br /><br /> Для создания профилей Wi-Fi и управления ими: **Политика автора**, **Изменение отчета**, **Чтение** и **Запуск отчета** для объекта **Профиль Wi-Fi**.<br /><br /> Роль безопасности **Диспетчер доступа к ресурсам компании** включает все разрешения, необходимые для управления профилями сертификатов в Configuration Manager. Дополнительные сведения см. в разделе **Настройка ролевого администрирования** статьи [Настройка безопасности в Configuration Manager](../../core/plan-design/security/configure-security.md).|
