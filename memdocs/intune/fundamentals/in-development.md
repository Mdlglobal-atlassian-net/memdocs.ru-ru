# <a name="in-development-for-microsoft-intune"></a>В разработке для Microsoft Intune

- Если мы полагаем, что вам потребуется принять меры до изменения, мы опубликуем дополнительное сообщение в Центре сообщений Office.

- Эта страница и страница [новых возможностей](whats-new.md) периодически обновляются. Проверьте наличие дополнительных обновлений.

## <a name="app-management"></a>Управление приложениями

### <a name="support-for-multiple-accounts-in-company-portal-for-mac---5779449----"></a>Поддержка нескольких учетных записей в приложении "Корпоративный портал" для Mac<!-- 5779449  -->
Приложение "Корпоративный портал" на устройствах macOS теперь кэширует учетные записи пользователей, упрощая вход в систему. Пользователям больше не нужно выполнять вход в приложение "Корпоративный портал" каждый раз при запуске приложения. Кроме того, в приложении "Корпоративный портал" будет отображаться средство выбора учетной записи, если кэшировано несколько учетных записей пользователей, поэтому пользователям не нужно вводить имя пользователя. 

### <a name="auto-update-vpp-available-apps---3640511----"></a>Автоматическое обновление доступных приложений VPP<!-- 3640511  -->
Приложения, опубликованные в качестве доступных приложений для программы Volume Purchase Program (VPP), будут автоматически обновляться, если для токена VPP включен параметр **Автообновление приложений**. Сейчас доступные приложения VPP не обновляются автоматически. Вместо этого конечные пользователи должны перейти в приложение "Корпоративный портал" и переустановить приложение, если доступна более новая версия. Однако требуемые приложения в настоящее время поддерживают автоматическое обновление.

### <a name="customize-self-service-device-actions-in-the-company-portal--4393379----"></a>Настройка действий самообслуживания устройств в приложении "Корпоративный портал"<!--4393379  -->
Вы сможете настроить доступные действия самообслуживания, которые отображаются для конечных пользователей в приложении "Корпоративный портал" и на веб-сайте. Чтобы помочь предотвратить непредвиденные действия устройств, можно настроить эти параметры для приложения "Корпоративный портал", установив флажок **Администрирование клиента** > **Настройка** > **Создать** > **Скрыть функции**. Доступны перечисленные ниже действия.
- Скрыть кнопку **Удалить** на корпоративных устройствах с Windows.
- Скрыть кнопку **Сброс** на корпоративных устройствах с Windows.
- Скрыть кнопку **Сброс** на корпоративных устройствах с iOS.
- Скрыть кнопку **Удалить** на корпоративных устройствах с iOS.

Дополнительные сведения см. в разделе [Действия для самообслуживания устройства, выполняемые пользователем из приложения "Корпоративный портал"](../apps/company-portal-app.md#user-self-service-device-actions-from-the-company-portal).

### <a name="unified-delivery-of-azure-ad-enterprise-or-office-online-applications-in-the-company-portal--4404429---"></a>Единая доставка приложений Azure AD Enterprise или Office Online в приложении "Корпоративный портал"<!--4404429 -->
Вы сможете переключить (**Скрыть** или **Показать**) отображение приложений Azure AD Enterprise или Office Online в приложении "Корпоративный портал". Каждый пользователь увидит весь каталог приложений из выбранной службы Майкрософт. По умолчанию для каждого дополнительного источника приложения будет установлено значение **Скрыть**. Эта функция сначала вступит в силу на веб-сайте "Корпоративный портал" в выпуске 2005 с поддержкой корпоративных порталов для Windows, iOS/iPadOS и macOS, которые должны были появиться вскоре. В [центре администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) выберите **Администрирование клиентов** > **Настройка**, чтобы найти этот параметр. Дополнительные сведения см. в разделе [Как настроить приложения Корпоративного портала Intune, веб-сайт "Корпоративный портал" и приложение Intune](../apps/company-portal-app.md).

### <a name="search-the-intune-docs-from-the-company-portal---1736480---"></a>Поиск документов Intune в приложении "Корпоративный портал"<!-- 1736480 -->
Теперь вы можете выполнять поиск в документации Intune непосредственно из приложения "Корпоративный портал" для macOS. В строке меню выберите **Справка** > **Поиск** и введите ключевые слова для поиска, чтобы быстро найти ответы на интересующие вас вопросы.

### <a name="company-portal-for-android-will-guide-users-to-get-apps-after-work-profile-enrollment----6103999----"></a>Корпоративный портал для Android позволяет пользователям получать приложения после регистрации рабочего профиля <!-- 6103999  -->
Мы улучшаем встроенное руководство по работе в приложении "Корпоративный портал", чтобы пользователи могли легко находить и устанавливать приложения.  После регистрации в управлении рабочим профилем пользователи увидят сообщение о том, что они могут найти рекомендуемые приложения в версии Google Play с индикатором. Пользователи также увидят новую ссылку **Получить приложения** на панели приложения "Корпоративный портал" слева. Чтобы использовать эти новые и улучшенные возможности, вкладка **Приложения** будет удалена. 

### <a name="android-company-portal-user-experience---5736084---"></a>Пользовательский интерфейс приложения "Корпоративный портал" для Android<!-- 5736084 -->
В выпуске 2005 приложения "Корпоративный портал" для Android конечные пользователи устройств Android, которые выдавали предупреждение, блокировали или очищали устройства с помощью политики защиты приложений, увидят новый интерфейс пользователя. Вместо текущего диалогового окна конечные пользователи увидят полноразмерное сообщение с описанием причины предупреждения, блокировки или очистки, а также действий по устранению проблемы. Дополнительные сведения см. в статьях [Возможности защиты приложений для устройств Android](../apps/app-protection-policy.md#app-protection-experience-for-android-devices) и [Параметры политики защиты приложений Android в Microsoft Intune](../apps/app-protection-policy-settings-android.md).

## <a name="device-configuration"></a>Конфигурация устройства

### <a name="device-configuration-profile-settings-and-values-will-be-updated-for-windows-platforms---4091122---"></a>Параметры и значения профиля конфигурации устройства будут обновлены для платформ Windows<!-- 4091122 -->
При создании профилей конфигурации устройств для платформ Windows (**Устройства** > **Профили конфигурации** > **Создать профиль** > любой вариант **Windows** в качестве платформы) некоторые параметры и их значения отличаются от CSP и могут сбивать с толку. Имена параметров и их значения будут обновлены, чтобы стать более понятными.

Область применения:

- Профили конфигурации устройств для Windows 10 и более поздних версий
- Профили конфигурации устройств с Windows Holographic for Business
- Профили конфигурации устройств Windows 8.1
- Профили конфигурации устройств Windows Phone 8.1

### <a name="configure-the-microsoft-defender-atp-app-for-macos-----5520115----"></a>Настройка приложения ATP в Microsoft Defender для macOS  <!-- 5520115  -->
Вскоре вы сможете настроить [параметры](../protect/endpoint-protection-macos.md) для приложения ATP в Microsoft Defender для устройств, на которых выполняется macOS в рамках профиля конфигурации устройств Endpoint Protection (**Устройства** > **Профили конфигурации** > **Создать профиль**, выберите **macOS** для параметра *Платформа*, а затем **Endpoint Protection** для параметра *Тип профиля*). Для конфигурации устройств macOS будет использоваться восемь параметров. 

ATP для Microsoft Defender поддерживается в macOS 10.13 (High Sierra) и более поздних версий, а приложение [ATP в Microsoft Defender](../apps/apps-advanced-threat-protection-macos.md) должно быть развернуто на устройстве *после* применения этих параметров. Перед развертыванием приложения необходимо отправить параметры на устройство. Бета-версии macOS не поддерживаются.

### <a name="new-filevault-setting-for-macos-endpoint-protection-device-configuration-policy---5459801-----"></a>Новый параметр FileVault для политики конфигурации устройств macOS Endpoint Protection<!-- 5459801   -->
Мы добавляем новый параметр в категорию FileVault в шаблоне [macOS Endpoint Protection](../protect/endpoint-protection-macos.md): Скройте ключ восстановления. (**Устройства** > **Профили конфигурации** > **Создать профиль**, выберите **macOS** для параметра *Платформа*, а затем **Endpoint Protection** для параметра *Тип профиля*). Этот параметр скрывает личный ключ от конечного пользователя во время шифрования FileVault 2. Пользователь устройства может просматривать свой личный ключ восстановления в любое время из приложения Корпоративного портала для iOS или с веб-сайта Корпоративного портала для зашифрованного устройства macOS. Чтобы просмотреть личный ключ восстановления, можно перейти к сведениям об устройстве и щелкнуть *Получить ключ восстановления*.

Этот параметр будет недоступен в ранее созданной политике. Чтобы настроить этот параметр для использования, необходимо повторно создать политики FileVault. 

### <a name="configure-system-extensions-on-macos-devices---6255624----"></a>Настройка системных расширений на устройствах macOS<!-- 6255624  -->
На устройствах macOS можно создать профиль расширений ядра для настройки параметров на уровне ядра (**Устройства** > **Профили конфигурации** > **macOS** для платформы > **Расширения ядра** для профиля). Компания Apple со временем отказывается от расширений ядра и заменяет их системными расширениями в будущем выпуске. Системные расширения запускаются в пространстве пользователя и не предоставляют доступ к ядру. Цель заключается в повышении безопасности и предоставлении большего контроля конечным пользователям, ограничивая атаки на уровне ядра. Как расширения ядра, так и системные расширения позволяют пользователям устанавливать расширения приложений, расширяющие собственные возможности операционной системы.

В Intune можно настроить как расширения ядра, так и системные расширения (**Устройства** > **Профили конфигурации** > **macOS** для платформы > **Системные расширения** для профиля). Расширения ядра применяются к версии 10.13.2 и более поздним версиям. Системные расширения применяются к версии 10.15 и более поздним версиям. От macOS 10.15 до macOS 10.15.4 расширения ядра и системные расширения могут выполняться параллельно. 

Дополнительные сведения о расширениях ядра на устройствах macOS см. в разделе [Добавление расширений ядра в macOS](../configuration/kernel-extensions-overview-macos.md).

Область применения:
- macOS 10.15 и более поздние версии;

### <a name="automated-device-sync-interval-down-to-12-hours--3077535---"></a>Интервал автоматической синхронизации устройства сокращен до 12 часов<!--3077535 -->
Для автоматической регистрации устройств Apple интервал автоматической синхронизации устройств между Intune и Apple Business Manager будет сокращен с 24 часов до 12 часов. Дополнительные сведения о синхронизации см. в разделе [Синхронизация управляемых устройств](../enrollment/device-enrollment-program-enroll-ios.md#sync-managed-devices).

### <a name="enrollment-restrictions-will-support-scope-tags--4209550---"></a>Ограничения регистрации будут поддерживать теги области<!--4209550 -->
Вы сможете назначить теги области для ограничений регистрации. Для этого перейдите на страницу [Центр администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) > **Устройства** > **Ограничения регистрации** > **Создать ограничение**. Создайте ограничение любого типа, и вы увидите страницу **Теги области**.

### <a name="shared-ipads-for-business--6367326---"></a>Общие iPad для бизнеса<!--6367326 -->
Вы сможете использовать Intune и Apple Business Manager для простой и безопасной настройки общего iPad, чтобы несколько сотрудников могли совместно использовать это устройство. [Общий iPad](https://developer.apple.com/education/shared-ipad/) Apple обеспечивает персонализированный интерфейс для нескольких пользователей, сохраняя данные пользователей. Используя управляемый идентификатор Apple ID, пользователи могут получать доступ к своим приложениям, данным и параметрам после входа на любой общий iPad в своей организации. Общий iPad поддерживает федеративные удостоверения.

Чтобы увидеть эту функцию, перейдите на страницу [Центр администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) > **Устройства** > **iOS** > **Регистрация устройств iOS** > **Токены программы регистрации** > выберите токен** > **Профили** > **Создать профиль** > **iOS**. На странице **Параметры управления** выберите **Регистрация без сходства пользователей**, чтобы увидеть параметр **Общий iPad**.

**Применимо к:** iPadOS 13.4 и более поздних версий. В этом выпуске добавлена поддержка временных сеансов с общим iPad, чтобы пользователи могли получить доступ к устройству без управляемого идентификатора Apple ID. После выхода пользователя устройство удаляет все его данные, чтобы немедленно быть готовым к использованию без необходимости в очистке.

## <a name="security"></a>Безопасность

### <a name="derived-credentials-support-for-disa-purebred-on-android-devices--4839592---"></a>Поддержка производных учетных данных для DISA Purebred на устройствах Android<!--4839592 -->
Вы можете использовать *DISA Purebred* в качестве поставщика [производных учетных данных](../protect/derived-credentials.md) на полностью управляемых устройствах Android Enterprise (**Администрирование клиента** > **Соединители и токены** > **Производные учетные данные**). Будет включена поддержка для получения производных учетных данных для DISA Purebred. Вы сможете использовать производные учетные данные для проверки подлинности приложения, подключения к сети Wi-Fi, подключения к сети VPN или для подписи S/MIME, а также для шифрования с приложениями, которые поддерживают такие учетные данные. 

В апреле Intune добавлена поддержка *Entrust Datacard* и *Intercede* в качестве поставщиков производных учетных данных. 

### <a name="privacy-preferences-settings-for-macos-devices---2934232---"></a>Параметры конфиденциальности для устройств macOS<!-- 2934232 --> 
С выпуском macOS Catalina 10.15 корпорация Apple добавила новые усовершенствования в области безопасности и конфиденциальности. По умолчанию приложения и процессы не могут получить доступ к конкретным данным без согласия пользователя. Если пользователи не предоставляют согласие, то приложения и процессы могут не работать. Intune добавляет поддержку параметров, которые позволяют ИТ-администраторам разрешать или запрещать согласие на доступ к данным от имени конечных пользователей на устройствах под управлением macOS 10.14 и более поздних версий. Эти параметры гарантируют, что приложения и процессы продолжат функционировать должным образом и будут отправлять конечным пользователям меньше запросов.


### <a name="duplicate-your-policies-in-endpoint-security---5892558-----"></a>Дублирование политик в системе безопасности конечных точек<!-- 5892558   -->
Вы сможете выбрать политику, созданную в узле безопасности конечной точки центра администрирования Microsoft Endpoint Manager, а затем дублировать ее, чтобы создать копию.  Ниже перечислены политики, которые вы сможете дублировать: 

- Антивирусная программа
- Шифрование дисков
- Брандмауэр
- Обнаружение и отклик конечной точки
- Уменьшение уязвимой зоны
- Защита учетной записи

После дублирования будет создана копия исходной политики, которую можно переименовать и изменить. Копия не будет содержать назначения из исходной политики.

### <a name="send-push-notifications-as-an-action-for-non-compliance----1733150-----"></a>Отправка push-уведомлений в качестве действия для несоответствия <!-- 1733150   -->
Для платформ iOS и Android мы добавляем новое действие для несоответствия, которое будет отправлять push-уведомление от приложения через приложение "Корпоративный портал". Пользователи могут нажимать на уведомления, чтобы запустить приложение "Корпоративный портал" и просмотреть причину несоответствия. Администраторы смогут настроить это новое действие для несоответствия в Центре администрирования Microsoft Endpoint Manager, перейдя в раздел **Устройства** > **Политики соответствия** > **Создать политику**, а затем выбрав *Действие*, чтобы отправлять push-уведомление от приложения. 

### <a name="new-profile-for-endpoint-security-firewall-policy---5653324-----"></a>Новый профиль для политики брандмауэра Endpoint Security<!-- 5653324   -->
В качестве предварительной версии мы добавляем дополнительный профиль для Windows 10 и более поздней версии в политику брандмауэра в безопасности конечной точки Intune (**Безопасность конечной точки** > **Брандмауэр** > **Создать политику** > выберите **Windows 10 и более поздние версии**). 

Каждый экземпляр этого нового профиля поддерживает до 150 пользовательских *правил брандмауэра Защитника Майкрософт*. Профиль правил брандмауэра Защитника Майкрософт позволяет определять детализированные правила брандмауэра Windows, разрешающие или блокирующие порты и приложения в Windows 10.
